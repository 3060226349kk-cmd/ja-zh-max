# 从教材到技能：并行子代理萃取工作流

本文档记录 ja-zh-translation-polish v2.0 升级过程中使用的教材萃取方法。
适用于：将大型翻译教材/参考书（200+ 页）系统融入翻译 skill 的场景。

## 工作流

### 第一步：PDF 文字提取

```bash
python3 -c "
import fitz
doc = fitz.open('教材.pdf')
with open('教材_提取.txt', 'w', encoding='utf-8') as f:
    for i in range(doc.page_count):
        f.write(f'\n=== 第{i+1}页 ===\n')
        f.write(doc[i].get_text())
"
```

若 PDF 无文字层（纯扫描件），使用 marker-pdf OCR：

```bash
# marker-pdf 需独立 venv（与 Hermes 依赖冲突）
~/.local/marker-venv/Scripts/marker 教材.pdf 输出文件夹 --output_format markdown
```

### 第二步：识别全书结构

从提取文本中找到目录，将全书按方法论价值分级：

| 优先级 | 章节类型 | 处理方式 |
|--------|---------|---------|
| ★★★ | 核心方法论（语法对比、翻译技巧、病症分析） | 全文精读，提取所有实例 |
| ★★ | 辅助章节（文体、修辞、修改法） | 提取核心观点 + 代表实例 |
| ★ | 教学框架（练习、参考译文、序言） | 略读，只取方法论亮点 |

### 第三步：并行派发子代理

将章节按内容量均分给 3 个子代理，每个子代理的任务：

```
任务: 从提取文本中精读指定章节，返回结构化摘要:
1. 每章小节标题
2. 核心方法论观点（带原文引用）
3. 所有翻译实例（日文原句 + 错误译法 + 正确译法）
4. 标注页码（=== 第N页 ===）
```

**关键**：context 中必须包含精确的文件路径、页码范围、输出格式要求。

### 第四步：集成升级

子代理返回后，将摘要中的内容注入 skill 文件：

1. **SKILL.md**：注入核心理念（带页码引用）→ 阶段 0–5 逐节升级
2. **症状清单**：新增病症（带原文引用 + 实例）
3. **技巧库**：新增技巧（带原文引用）
4. **文本分析**：补充理论框架
5. **新建参考文件**：如修改三层法等独立方法论模块

### 第五步：版本升级

- 版本号 +1 大版本
- 所有新内容带「（高宁，第X章，第Y页）」格式的页码引用
- 跑标点脚本验证
- 清理中间文件（提取文本、子代理摘要、临时脚本）
